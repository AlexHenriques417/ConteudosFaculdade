AULA I - ESTRUTURAS DE DADOS - UNIDADE I

- PSEUDOCÓDIGO 
É UM ALGORITMO COM LINGUAGEM INFORMAL 
UTILIZADA PARA PLANEJAR ALGORITMO
NÃO SEGUE SINTAXE RÍGIDA DE PROGRAMAÇÃO

- OBJETIVO 
AJUDA NA ORGANIZAÇÃO LÓGICA DAS IDEIAS
DESCREVE ALGORITMOS DE FORMA SIMPLES

- LINGUAGENS
ALTO NÍVEL - PRÓXIMA DA LINGUAGEM HUMANA ( LER E ESCRITA )
BAIxO NÍVEL - PRÓXIMAS DA LINGUAGEM DE MÁQUINAS COM CÓDIGO DIFICIL DE LER E ESCREVER

- CONCENTRE NA LÓGICA DE ALGORITMO 
SEM DETALHES DE IMPLEMENTAÇÃO
PONTE ENTRE LINGUAGEM HUMANA E LINGUAGEM DE MÁQUINA

- VARIÁVEIS SÃO CRIADAS JÁ SENDO INICIALIZADAS E SEM O TIPO
SOMA = O
MULTIPLICAÇÃO = 1
PALAVRA = " "

- ELEMENTO NEUTRO 
É POSSIVEL INICIALIZAR VARIÁVEL SEM ATRIBUIÇÃO 
QUANDO A VARIÁVEL NÃO PUDER TER VALOR INICIAL E PRECISA CAPTAR DO TECLADO

- VETORES ( ARRAY UNIDEMENSIONAIS ) -> LINHA ÚNICA/ SEQUÊNCIA LINEAR/ ÚNICO ÍNDICE
SEQUÊNCIA DE DADOS USADOS PARA ARMAZENAR EM UMA ÚNICA LINHA E ACESSAR POR INDICES

- MATRIZES ( ARRAY BIDIMENSIONAIS ) -> DUAS DIMENSÕES ( LINHA E COLUNA )
TABELA DE ELEMENTOS COM LINHAS E COLUNAS, UTILIZA DOIS ÍNDICES PARA ACESSAR DADOS

- AAA[10] - QUANTIDADE DE ELEMENTOS NOS VETORES
- BBB[5][7] - LINHAS E COLUNAS DA MATRIZ

- O PRIMEIRO ELEMENTO COMEÇA NA POSIÇÃO 1 TANTO NO VETOR QUANTO NA MATRIZ

- EXEMPLO 1 ( PROGRAMA CONVECIONAL ) 

INICIO 
  LEIA (NUMERO1)
  LEIA (NUMERO2)
  LEIA (NUMERO3)
  SOMA = NUMERO1 + NUMERO2 + NUMERO3
  MEDIA = SOMA / 3
  ESCREVA ("A MÉDIA É: " + MEDIA )
FIM

OBS - O " + " SERVE PARA CONCETENAR TEXTOS E VARIÁVEIS


- EXEMPLO 2 ( FLUXO DE DECISÃO )

SE X = 1
  ESCREVA ("UM")
  ESCREVA ("FIM DO PROGRAMA")
SENÃO
  ESCREVA ("NÃO É UM")
  ESCREVA ("FIM DO PROGRAMA")
FIM SE

OBS - O BLOCO É DEFINIDO ATRAVÉS DA IDENTAÇÃO
OBS - CASP PRECISE FAZER UM OUTRO SE PRECISA SER COLOCADO DENTRO DO SENÃO


- EXEMPLO 3 ( FLUXO DE REPETIÇÃO )

- O "PARA" EQUIVALE O "FOR"
- USADO QUANDO SOUBER A QUANTIDADE DE REPETIÇÕES FORAM FEITAS
- É ATUALIZADA AUTOMATICAMENTE
- INCREMENTO EM 1 EM 1

SOMA = 0
N = 10
PARA X VARIANDO DE 1 ATÉ N
  SOMA = SOMA + X
ESCREVA ( SOMA )


- EXEMPLO 4 ( FLUXO DE REPETIÇÃO )

- O "ENQUANTO" EQUIVALE O "WHILE"
- USADO PARA FAZER TESTES
- CASO O TESTE SEJA VERDADEIRO ELE EXECUTA UM TRECHO 
- A CONDIÇÃO SERÁ ATUALIZADA DENTRO DO CORPO DO "ENQUANTO"

VAR = 0
ENQUANTO VAR <= 10
  VAR = VAR + 1
FIM ENQUANTO //OPTATIVO
ESCREVA ("VAR MAIOR QUE 10")


- EXEMPLO 5 ( FLUXO DE REPETIÇÃO )

- O "FAÇA...ENQUANTO" EQUIVALE O "DO...WHILE"
- USADO QUANDO PRECISA EXECUTAR UM TRECHO DE CÓDIGO PRIMEIRO E DEPOIS FAZ O TESTE
- A CONDIÇÃO SERÁ ATUALIZADA DENTRO DO CORPO DO "FAÇA...ENQUANTO"

VAR = 0
FAÇA 
  VAR = VAR + 2
ENQUANTO VAR < 20
ESCREVA ("FINAL: " + VAR )


- EXEMPLO 6 ( FUNÇÕES )

- SÃO CRIADAS APENAS COOM SEU NOME, SEM O TIPO
- RECEBE O PARÂMETRO DENTRO DOS ( )
- A FUNÇÃO RETORNA O ÚNICO VALOR ATRAVÉS DO COMANDO "RETORNE"

SOMA ( A,B )
  TOTAL = A + B 
  RETORNE TOTAL 
FIM // USO OPTATIVO

INÍCIO
  LEIA ( X )
  LEIA ( Y )
  Z = SOMA ( X,Y )
  ESCREVA ( Z )
FIM
