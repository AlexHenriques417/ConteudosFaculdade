AULA I - ESTRUTURAS DE DADOS - UNIDADE II

- ESTRUTURAS DE DADOS LINEARES
POSSUI UM PREDECESSOR ( MÁXIMO )
POSSUI UM SUCESSOR ( MÁXIMO )
ORGANIZADOS EM ORDEM
INSERDOS OU REMOVIDOS
LISTAS, FILHAS, PILHAS E DEQUES

- LISTAS 
ENCADEADAS OU DUPLAMENTE ENCADEADAS
CADA NÓ GUARDA SEU CONTEÚDO 
GUARDA A INFORMAÇÃO DO PRÓXIMO
O ÚLTIMO SERÁ O NULL

- LISTAS DUPLAMENTE ENCADEADAS
CADA NÓ GUARDA SEU CONTEÚDO
GUARDA A INFORMAÇÃO DO NÓ PREDECESSOR E SUCESSOR
OPERAÇÕES DE BUSCA
INSERÇÃO, DELEÇÃO E ORDENAÇÃO

- LISTAS CIRCULARES
TODOS OS NÓS TEM PREDECESSOR E SUCESSOR

- FILHAS
FIRST IN OU FIRST OUT ( LIFO )
O PRIMEIRO ELEMENTO A ENTRAR NA FILA É O PRIMEIRO A SAIR
UTILIZADO PARA GERENCIAR TAREFAS E CONTROLE DE DOCUMENTOS

- PILHAS
LAST IN OU FIRST OUT ( LIFO )
O ÚLTIMO ELEMENTO A ENTRAR É O PRIMEIRO A SAIR
USADAS PARA ALOCAÇÃO E LIBERAÇÃO DE MEMÓRIA
OPERAÇÕES DE RECURSIVIDADE SÃO FEITAS EM PILHAS

- DEQUES
DOUBLE ENDED QUEUE
FILAS DUPLAMENTE TERMINADAS
ELEMENTOS ADICIONADOS OU RETIRADOS
EM QUALQUER EXTREMO ( CABEÇA OU CAUDA )
UTILIZADOS EM SISTEMAS DISTRIBUIDOS

- ESTRUTURAS NÃO LINEARES
ÁRVORES - BINÁRIAS OU DE BUSCA
HEAP ( FILAS DE PRIORIDADE ) - REMOVER O OBJETO COM PRIORIDADE
GRAFOS - PONTOS QUE APRESENTAM RELAÇÕES ENTRE SISTEMAS

- ÁRVORES
NÃO É LINEAR, NEM SEQUENCIAL E NEM ENFILEIRADA
DADOS ACIMA OU EM BAIXO DE OUTROS DADOS
NÓS ( VÉRTICES ) QUE GUARDAM VALORES
ARCOS ( ARESTAS ) LIGAM OS NÓS
NÓS SEM FILHOS SÃO CHAMADOS DE FOLHAS

- ÁRVORES ( REPRESENTAÇÕES )
DIAGRAMA DE INCLUSÃO
DIAGRAMA DE BARRAS
NUMERAÇÃO EM NÍVEIS
REPRESENTAÇÃO ANINHADA

- ÁRVORES ( CONCEITOS )
NÓS >= 0
SE NÓS = 0 -> ESTÁ VAZIA
O PRIMEIRO NÓ É A RAIZ
SUBÁRVORES - QUEM VEM DEPOIS DA RAIZ
FILHOS SÃO NÓS DAS SUBÁREVORES ( QUE HERDOU DA RAIZ )
GRAU É O NÚMERO DE FILHOS DE UM NÓ
FOLHA É UM NÓ COM GRAU ZERO
NÓ NÃO TERMINAL NÃO É FOLHA, MAS TAMBÉM NÃO É A RAIZ DA ÁRVORES, SÃO CHAMADOS DE GALHOS

- CARACTERISTICAS
UM NÓ FILHO NÃO PODE TER DOIS PAIS
O FILHO NÃO SABE QUEM É O SEU PAIS

( QUANDO ISSO ACONTECER É UM GRAFO )

- ÁRVORES ( HIERARQUIA )
TIPO DE RELACIONAMENTO COM SUBCONJUNTOS DE COMPONENTES É SUBORDINADO A OUTROS

- TIPOS DE HIERARQUIA
ESPECIALIZAÇÃO
COMPOSIÇÃO
DEPENDÊNCIA

- ABSTRAIR
CAPACIDADE MENTAL DE ANALISAR ALGO QUE EXISTE NO MUNDO REAL
CONSIDERAR SUAS CARACTERISTICAS, DESPREZANDO OS DETALHES

- INSTÂNCIA
ALGO QUE EXISTE NO MUNDO REAL
É IDENTIFICADA DE FORMA ÚNICA

- ESPECIALIZAÇÃO
RELACIONAMENTOS DO TIPO "É UM" OU "É UMA"
GRUPO COM A MESMA INSTÂNCIA ( CARACTERISTICAS IDÊNTICAS )
A ENTIDADE FILHA CONTINUA TENDO TODAS AS CARACTERISTICAS DA ENTIDADE MÃE

- INVERSO DA ESPECIALIZAÇÃO É A GENERALIZAÇÃO
PEGA CARACTERISTICAS EM COMUM EM ENTIDADES DIFERENTES E CRIA ENTIDADE MAIS GERAL COM ESSES DADOS

- COMPOSIÇÃO
CONJUNTO DAS PARTES ( TODO )
TODO É FORMADO PELA UNIÃO DAS PARTES
AO JUNTAS TODAS AS ENTIDADES ( PARTES ) É POSSIVEL CRIAR UMA ENTIDADE MAIOR
RELACIONAMENTO "TEM UM", "TEM UMA", "É FEITO DE", "É PARTE DE"

- INVERSO DA COMPOSIÇÃO É A DECOMPOSIÇÃO
AO FRANCIONAR O TODO É POSSIVEL OBTER AS SUAS PARTES

- AGREGAÇÃO 
AS PARTES EXISTEM SEM O TODO E O TODO EXISTE SEM AS PARTES ( CADA UM É INDEPENDENTE )

- COMPOSIÇÃO
PARTE NÃO EXISTE SEM O TODO ( DEPENDÊNCIA )
