AULA IV - ESTRUTURAS DE DADOS

- FOR
COMPLEXIDADE N + 1 
PRECISA DE 1 PARA TESTAR E DA ERRO
AS DEMAIS LINHAS DENTRO DO FOR SÃO N

- ATRIBUIÇÃO
LINHAS DEVEM POSSUIR ATRIBUIÇÃO ( PARA TER COMPLEXIDADE )
NUNCA TERÁ ATRIBUIÇÃO NA ASSINATURA -> LINHA 1
TODO FOR TEM ATRIBUIÇÃO
WHILE SÓ TEM TESTES E NÃO ATRIBUIÇÃO

- IMPORTANTE
QUEM TA DENTRO DOR FOR TERÁ A MESMA COMPLEXIDADE
WHILE SÓ TEM COMPARAÇÕES
COMEÇA DO PIOR CASO - ORDEM DECRESCENTE
INSERÇÃO DE ORDEM - CRESCENTE - VETOR COLOCADO EM ORDEM
N - TAMANHO DO VETOR
QUANTIDADE DE ELEMENTOS

- ATUANDO
QUANDO J = N SERÁ N - 1
QUANDO O I = 0 SERÁ, ELE IRÁ FALHAR
1 + 2 + 3 + 4 + ... + N + 1 --> N(N-1) / 2

- TIPOS DE FUNÇÕES
QUANDO FOR WHILE --> N^2 + 3N - 3
QUANTIDADE DE EXECUÇÕES DE CADA LINHA --> 3N^2 + 7N - 8
TEMPO --> CN^2 + CN + C

- LEMBRETE
A LINHA DO WHILE -> N(N-1) / 2 + (N-1)
A MAIOR GRANDEZA É O N^2 -> POIS ELEVA O VALOR
TERMOS MENORES SERÃO DESCARTADOS
MAIORES GRANDEZAS = N^2 / 3N^2 / CH^2
