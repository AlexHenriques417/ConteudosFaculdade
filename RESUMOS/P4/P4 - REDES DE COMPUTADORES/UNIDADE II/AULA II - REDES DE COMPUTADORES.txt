AULA II - REDES DE COMPUTADORES - UNIDADE II

- APLICAÇÕES P2P
SEM SERVIDOR SEMPRE LIGADO
SISTEMAS FINAIS QUE SE COMUNICAM DIRETAMENTE
PARES SOLICITAM SERVIÇO DE OUTROS PARES

- AUTO ESCALABILIDADE
NOVOS PARES TRAZEM NOVA CAPACIDADE DE ATENDIMENTO E NOVAS DEMANDAS DE SERVIÇOS

- ARQUITETURA PEER-TO-PEER
OS PARES ESTÃO INTERMITENTEMENTE CONECTADOS E ALTERAM ENDEREÇOS IP

- TEMPO DE DISTRIBUIÇÃO DE ARQUIVOS : CLIENTE-SERVIDOR
TEMPO PARA DISTRIBUIR F PARA N CLIENTES USANDO ABORDAGEM CLIENTE-SERVIDOR

- TRANSMISSÃO DO SERVIDOR 
DEVE ENVIAR SEQUENCIALMENTE N CÓPIAS DO ARQUIVOS

- CLIENTE 
CADA CLIENTE DEVE BAIXAR CÓPIA DO ARQUIVO

- TEMPO DE DISTRIBUIÇÃO DE ARQUIVOS : P2P
TEMPO PARA DISTRIBUIR F PARA OS N CLIENTES USANDO ABORDAGEM P2P 

- TRANSMISSÃO DO SERVIDOR 
DEVE ENVIAR PELO MENOS UMA CÓPIA

- CLIENTE 
CADA CLIENTE DEVE BAIXAR CÓPIA DO ARQUIVO

- CLIENTES
AGREGADOS DEVEM BAIXAR NF BITS

- DISTRIBUIÇÃO DE ARQUIVOS P2P : BITTORRENT
ARQUIVO DIVIDIDO EM PEDAÇÕS DE 256 KB
PARES EM TORRENT ENVIAM/ RECEBEM PEDAÇOS DE ARQUIVOS
DURANTE O DOWNLOAD, PAR ENVIA PEDAÇOS PARA OUTROS PARES
PAR PODE MUDAR DE PARES COM QUEM TROCA PEDAÇOS
PARES PODEM IR E VIR
O PAR QUE TIVER O ARQUIVO INTEIRO PODE SAIR OU PERMANECER NO TORRENT

- TRACKER( RASTREADOR )
RASTREIA PARES PARTICIPANDO DO TORRENT

- TORRENT
GRUPO DE PARES TROCANDO PEDAÇOS DE UM ARQUIVO

- SOLICITANDO PEDAÇOS 
QUALQUER MOMENTO, DIFERENTES PARES TEM SUBCONJUNTO DIFERENTES DE PEDAÇOS DE ARQUIVO
ALIC PEDE A CADA PAR PARA QUE ENVIEM UMA LISTA DOS PEDAÇOS QUE ELAS TEM
ALICE PEDE PEDAÇOS FALTANTES PARA OS PARES, OS MAIS RAROS PRIMEIRO

- ENVIANDO PEDAÇOS 
ALICE ENVIA PEDAÇOS PARA OS QUATRO PARES QUE ATUALMENTE ESTÃO ENVIANDO SEUS PEDAÇOS A UMA TXA MAIS ALTERAM
A CADA 30 SEGUNDOS, SELENCIONA ALEATORIAMENTE OUTRO PAR, COMEÇA A ENVIAR PEDAÇOS

- IMPORTANTE
MAIOR TAXA DE UPLOAD : ENCONTRA MELHORES PARCEIUROS, OBTEM ARQUIVO MAIS RÁPIDO
