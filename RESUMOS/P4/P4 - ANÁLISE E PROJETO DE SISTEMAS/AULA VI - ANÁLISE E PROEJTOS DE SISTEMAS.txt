AULA VI - ANÁLISE E PROEJTOS DE SISTEMAS

- INTRODUÇÃO
O DIAGRAMA DE CASOS DE USO MOSTRA O COMPORTAMENTO EXTERNO DO SISTEMA
É O MAIS ABSTRATO, FLEXÍVEL E INFORMAL DA UML
UTILIZADO NAS ETAPAS DE LEVANTAMENTO E ANÁLISE DE REQUISITOS
MODIFICADO DURANTE TODO O PROCESSO DE ENGENHARIA
SERVE DE BASE PARA OUTROS DIAGRAMAS
PODE SER USADO COMO PROTÓTIPO PARA CLIENTES
FACILITA A ORGANIZAÇÃO DOS REQUISITOS
MOSTRA O SISTEMA PELA PERSPECTIVA DO USUÁRIO
OBJETIVO: VISÃO EXTERNA DAS FUNÇÕES E SERVIÇOS QUE O SISTEMA DEVE OFERECER
DESCREVE O QUE O SISTEMA FAZ, MAS NÃO COMO

- QUANDO UTILIZAR
FERRAMENTA PARA ENTENDER REQUISITOS FUNCIONAIS
PRIMEIRA VERSÃO LOGO NO INÍCIO DO DESENVOLVIMENTO
DETALHES DEVEM SER ADICIONADOS POSTERIORMENTE
NÃO DEVE SER TORNADO EXCESSIVAMENTE COMPLEXO

- BENEFÍCIOS
FOCO NO USUÁRIO DO SISTEMA
FACILIDADE DE ENTENDIMENTO
AJUDA NO ACORDO ENTRE STAKEHOLDERS

- CASO DE USO
REQUISITO FUNCIONAL DO SISTEMA
SEQUÊNCIA DE INTERAÇÕES ENTRE SISTEMA E ATORES
DESCRIÇÃO TEXTUAL DAS FUNCIONALIDADES (CENÁRIOS)
ESPECIFICA VÁRIOS CENÁRIOS (EX.: RETIRAR DINHEIRO)
ASSOCIADO A UMA OU MAIS TELAS
REPRESENTADO POR ELIPSES (VERBO + OBJETO)
EXEMPLOS: “CADASTRAR CLIENTES”, “SACAR DINHEIRO”, “VERIFICAR PEDIDO”

- DIAGRAMA DE CASOS DE USO
FACILITA ENTENDIMENTO MOSTRANDO VISÃO EXTERNA.
MODELA O CONTEXTO DE UM SISTEMA OU SUBSISTEMA.
DELIMITA O ESCOPO E AS FUNCIONALIDADES.

- ELEMENTOS DO DIAGRAMA
ATORES → PAPÉIS DE ELEMENTOS EXTERNOS (PESSOAS, HARDWARE, OUTROS SISTEMAS)
CASOS DE USO → FUNCIONALIDADES DO SISTEMA
RELACIONAMENTOS:
ASSOCIAÇÃO
GENERALIZAÇÃO
EXTENSÃO (<<EXTENDS>>)
INCLUSÃO (<<INCLUDE>>)
FRONTEIRA DO SISTEMA → DELIMITA A ÁREA DE ATUAÇÃO.

- RELACIONAMENTOS
ASSOCIAÇÃO: LIGA ATORES A CASOS DE USO
GENERALIZAÇÃO: HERANÇA ENTRE ATORES OU CASOS DE USO
EXTENSÃO: COMPORTAMENTOS OPCIONAIS, EXECUTADOS EM CERTAS SITUAÇÕES
INCLUSÃO: COMPORTAMENTOS OBRIGATÓRIOS, USADOS EM MAIS DE UM CASO DE USO

- ERROS FREQUENTES
FAZER CASOS DE USO PEQUENOS DEMAIS (EX.: INCREMENTAR CONTADOR)
FAZER CASOS DE USO GRANDES DEMAIS (EX.: REALIZAR COMPRA)
UM CASO DE USO DEVE GERAR RESULTADO PERCEPTÍVEL PARA O ATOR
