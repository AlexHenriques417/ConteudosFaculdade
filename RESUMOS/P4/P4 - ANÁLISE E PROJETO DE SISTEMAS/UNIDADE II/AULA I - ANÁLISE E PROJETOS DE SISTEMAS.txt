AULA I - ANÁLISE E PROJETOS DE SISTEMAS - UNIDADE II

- INTRODUÇÃO
CONJUNTO DE CLASSES, INTERFACES E RELACIONAMENTOS DO SISTEMA
DIAGRAMA CENTRAL É MODELADO ORINETADO A OBJETO
VISÃO ESTÁTICA = CLASSES ESTÃO ORGANIZADAS
CLASSES ESPECIFICAM TANTO AS PROPRIEDADES QUANTO OS COMPORTAMENTOS DOS OBJETOS

- ESTRUTURA DA CLASSE
NOME DA CLASSE -> OBRIGATÓRIO
PROPRIEDADES -> OPCIONAL
OPERAÇÕES -> OPCIONAL

- FORMAS DE REPRESENTAÇÃO
NOME
NOME + PROPRIEDADES
NOME + OPERAÇÕES
NOME + PROPRIEDADES + OPERAÇÕES

- VIZUALIZAÇÃO DA CLASSE
NOME DO ATRIBUTO
TIPO DO DADO
MULTIPLICIDADE
VALOR DEFAULT
RESTRIÇÃO
TIPO DE RETORNO
NOME DA OPERAÇÃO
ATRIBUTO ESTÁTICO
VISIBILIDADE

- ELEMENTOS DE UM DIAGRAMA DE CLASSES
CLASSES
    ATRIBUTOS
    MÉTODOS
RELACIONAMENTOS
    ASSOCIAÇÃO
        AGREGAÇÃO
        COMPOSIÇÃO
    GENERALIZAÇÃO
    DEPENDÊNCIA

- CLASSES
REPRESENTADAS POR RETÂNGULOS
INCLUI NOME, ATRIBUTO E MÉTODOS
RECEBE NOME DE ACORDO COM O PROBLEMA

- ATRIBUTOS 
REPRESENTAM O CONJUTNTO DE CARACTERISTICAS DOS OBJETOS

- VISIBILIDADE
+ PÚBLICO -> VISÍVEL EM QUALQUER CLASSE DE QUALQUER PACOTE
# PROTEGIDO -> VISÍVEL PARA CLASSES DO MESMO PACOTE
- PRIAVDO -> VISÍVEL SOMENTE PARA CLASSE

- NOTAÇÃO COMPLETA DOS ATRIBUTOS
NOME -> NOME DO ATRIBUTO
TIPO -> DOMÍNIO DO ATRIBUTO
MULTIPLICIDADE -> INDICAÇÃO DE QUANTOS OBJETOS PODEM PREENCHER A PROPRIEDADES
VALOR DEFAULT -> VALOR DO ATRIBUTO, CASO ELE SEJA OMITIDO NO MOMENTO DA CRIAÇÃO
RESTRIÇÃO -> INIDCAR PROPRIEDADES ADICIONAIS

- MÉTODOS
REPRESENTAM O CONJUNTO DE OPERAÇÕES ( COMPORTAMENTOS ) QUE A CLASSE FORNECE

- NOTAÇÃO COMPLETA DOS MÉTODOS
NOME -> NOME DA OPERAÇÃO
LISTA DE PARÂMETROS -> PARÂMETROS DA OPERAÇÃO
MULTIPLICIDADE -> INDICAÇÃO DE QUANTOS OBJETOS PODEM PREENCHER A PROPRIEDADE
TIPO DE RETORNO -> TIPO RETORNADO PELA OPERAÇÃO
RESTRIÇÃO -> PERMITE INDICAR PROPRIEDADES ADICIONAIS

- OPERAÇÕES
OPERAÇÕES ABSTRATAS -> NÃO POSSUEM UMA IMPLEMENTAÇÃO ESPECIFICA ( ESCRITAS EM ITALICO )
OPERAÇÕES ESTÁTICAS -> NÃO PODEM SER SOBRECARREGADAS ( ESCRITA COM LINHA SUBLINADA )
OPERAÇÃO -> SERVIÇO SOLICITADO DE UMA CLASSE
MÉTODO -> IMPLEMENTAÇÃO CONCRETA DA OPERAÇÃO

- RELACIONAMENTOS
NOME -> DESCRIÇÃO DADA AO RELACIONAMENTO ( FAZ, TE, POSSUI )
SENTIDO DE LEITURA
NAVEGABILIDADE -> INIDICADA POR UMA SETA NO FIM DO RELACIONAMENTO
MULTIPLICIDADE -> 0...1, 0...*, 1, 1...*, 2, 3...7
TIPO -> ASSOCIAÇÃO ( AGREGAÇÃO, COMPOSIÇÃO), GENERALIZAÇÃO E DEPENDÊNCIA
PAPÉIS -> DESEMPENHOS POR CLASSES EM UM RELACIONAMENTO

- ASSOCIAÇÃO
INDICA QUE OS OBJETOS DE UMA CLASSE ESTÃO VINCULADOS A OBJETOS DE OUTRA CLASSE
REPRESENTADA POR UMA LINHA SÓLIDA CONECTANDO DUAS CLASSES
PODE SER UNIDIRECIONAL OU BIDIRECIONAL
AUTO ASSOCIAÇÃO - RELACIONA A CLASSE CONSIGO MESMA

- INDICADORES DE MULTIPLICIDADE
1 - EXATAMENTE UM
1...* - UM OU MAIS
0...* - ZERO OU MAIS ( MUITOS )
* - ZERO OU MAIS ( MUITOS )
0...1 - ZERO O UM
M...N - FAIXA DE VALORES ( EX: 4...7 )

- AGREGAÇÃO E COMPOSIÇÃO
"TODO-PARTE"
AGREGAÇÃO -> A "PARTE" EXISTE SEM O "TODO"
COMPOSIÇÃO -> O "TODO" CONTROLA O CICLO DE VIDA DA "PARTE" E ELA NÃO PODE SER COMPARTILHADA EM OUTROS RELACIONAMENTOS

- GENERALIZAÇÃO
RELACIONAMENTO ENTRE ITENS GERAIS E ITENS ESPECÍFICOS

- DIAGRAMA DE ATIVIDADES
DESCREVE A LÓGICA DE PROCEDIMENTOS, PROCESSOS DE NEGÓCIO E FLUXOS DE TRABALHO
PERMITE QUE SEJA MOSTRADO QUE ENTIDADE É RESPONSÁVEL POR CADA AÇÃO NO DIAGRAMA

- ELEMENTOS DO DIAGRAMA
ESTADOS INICIAS E FINAIS
ATIVIDADES E TRANSIÇÕES
DECISÕES
BIFURCAÇÃO E UNIÃO
RAIAS  

- ESTADOS INICIAIS E FINAIS
INDICA O INICIO DO PROCESSO
INDICA O FIM DO PROCESSO

- ATIVIDADES
AÇÃO QUE DEVE SER FEITAS
QUANDO FINALIZADAS TRANSFERE A EXECUÇÃO PARA A PRÓXIMA ATIVIDADE
CADA CAIXA É UMA AÇÃO, PASSOS DA ATIVIDADE

- TRANSIÇÕES
SETAS CONTÍNUAS QUE REPRESENTAM FLUXO DE TRABALHO DE UMA ATIVIDADE PARA A OUTRA

- DECISÕES
LOSANGO UTILIZADO PARA CONTROLAR OS DESVIOS DO FLUXO DE CONTROLE
RAMIFICAÇÃO -> UMA ENTRADA E DUAS SAÍDAS
MESCLAR -> DUAS ENTRADAS E UMA SÁIDA

- BIFURCAÇÃO E UNIÃO
BARRA SÓLIDA USADA PARA ATIVIDADES PARALELAS
BIFURCAÇÃO -> DIVISÃO DO FLUXO DE CONTROLE
UNIÃO -> SICRONIZAÇÃO DAS ATIVIDADES

- RAIAS
FORMA DE ORGANIZAÇÃO LÓGICA DAS ATIVIDADES
PODEM ESTAR ASSOCIADAS A OBJETOS, COMPONENTES DO SISTEMA OU A DOS ATORES
PARTICIPAM SUAS AÇÕES EM GRUPOS E CADA GRUPO REPRESENTA UMA UNIDADE DE NEGÓCIO POR EXECUTAR AS AÇÕES CONTIDAS DENTRO DAQUELA RAIAS

- SINAL 
PONTO DE PARTIDA CLARAMENTE DEFINIDO
UM SINAL DE TEMPO OCORRE DEVIDO Á PASSAGEM DO TEMPO
UM SINAL INDICA QUE A ATIVIDADE RECEBE UM EVENTO DE UM PROCESSO EXTERNO
